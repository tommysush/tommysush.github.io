Swift 存取資料的方式

PI of Lab3F 二月 26, 2021 Self-Taught

UserDefaults

App’s directory in device storage

當儲存的資料可能隨著 app 的使用而增加時，iOS 提供了簡便的方式讓我們讀取和寫入資料到裝置的儲存空間，也就是讓個別 app 都擁有一個獨立的目錄（directory）來存放。如果要存取該目錄時，必須要知道其正確的名稱，這時就需要 FileManager class 和 其方法urls(for:in:)的協助。

class FileManager: NSObject
func urls(for directory: FileManager.SearchPathDirectory, in domainMask: FileManager.SearchPathDomainMask) -> [URL]

首先，利用urls(for:in:)來創建 function ，以便找到所要的目錄。注意！在此假設我們的 app 所存取的資料為文件，因此，第一個參數 SearchPathDirectory 的值選用 documentDirectory ，第二個參數 SearchPathDomainMask 的值選用 userDomainMask 。由於urls(for:in:)會回傳一個 URL 的陣列，而陣列的 第一個值 就是我們所要找的文件目錄。

struct ContentView: View {
    var body: some View {
        // more code
    }

    func getDocumentsDirectory() -> URL {
        let paths = FileManager.default.urls(for: .documentDirectory, in: .userDomainMask)
        return paths[0]
    }
}

既然知道文件存放的目錄，接著只要應用到 app 需要寫入(write(to:))和讀取（String(contentsOf:)、Data(contentsOf:)）的地方即可。

var body: some View {
    Text("Tap here for saving and reading.")
        .onTapGesture {
            let str = "This line has been saved."
            let url = getDocumentDirectory().appendingPathComponent("saved.txt")

            do {
                try str.write(to: url, atomically: true, encoding: .utf8)
                let input = try String(contentsOf: url)
                print(input)
            } catch {
                print(error.localizedDescription)
            }
        }
}
