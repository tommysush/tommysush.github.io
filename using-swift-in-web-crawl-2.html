使用 Swift 進行網路爬蟲

PI of Lab3F 四月 30, 2022 Swift & SwiftUI

在進入正題之前，應該先對 Swift 的 URL Loading System、以及本篇即將使用的第三方套件 Alamofire 與 SwiftSoup 有些基本的認識，以下將先對這三個項目做簡單的介紹。

🔰 Foundation framework > URL Loading System

URL Loading System 讓使用者透過標準（e.g. https）的或自訂的協定（protocol）來取用以 URLs 識別的資源，而且是以非同步的（asynchronously）方式進行， 因此我們能在處理收到的 data 或 errors 的同時，保持 app 的回應而不需停頓。

簡單來說，必須使用 URLSession 的實例（instance）來創造一個或多個 URLSessionTask 的實例，以用於處理或回傳資料、下載檔案、甚至上傳資料或檔案給遠端。

URLSession 共有四種類型，其中符合簡易需求的 shared session 是不用創造而可直接使用的，至於其他三種則需自訂 URLSessionConfiguration 和 URLSessionDelegate 來創造。

  • shared session：符合一般需求使用的 request，沒有 configuration 物件。
  • default session：類似 shared，但能自訂 configuration 和 delegate。
  • ephemeral session：類似 shared，但不會將 caches、cookies、或 credentials 寫入磁碟。
  • background session：當 app 不在 running 狀態時，可在背景執行下載和上傳。

URLSessionTask 同樣也有四種類型，其作用不外乎從伺服器端接收或上傳資料，差別在於是以磁碟上的檔案（file）的形式、抑或是以記憶體中的 NSData 物件的形式。

  • dataTask：以 NSData 形式將伺服器的回應回傳到記憶體中。支援上述的前三種 session 類型，但不支援 background session。
  • uploadTask：類似 dataTask，但它也能送出資料（通常是磁碟上檔案的形式）至遠端。另外，此類型支援 background session。
  • downloadTask：下載資源為檔案形式於磁碟上，同樣支援background session。
  • webSocketTask：透過 TCP 與 TLS 交換訊息。

與其他網路 APIs 一樣，URLSession API 也是高度非同步化（asychronicity）。它透過以下兩種方式之一回傳資料，端看你呼叫的是那一種。

  • completion handler block：當傳輸成功地完成或回報錯誤（error）時呼叫 handler。
  • methods on the session's delegate：當資料抵達或者傳輸完成時呼叫。

注意下面這個實作範例就是以常用的 shared session 和 dataTask 來讀取網頁（台灣銀行當日匯率）內容到終端機（terminal）畫面裡。 注意！程式碼最後的 resume() 是絕對不能少的。

import SwiftUI

struct ContentView: View {
    var body: some View {
        Text("Hello, world!")
            .onTapGesture { self.startLoad() }
    }
    
    func startLoad() {
        // use guard let statement to ensure url isn't an optional
        guard let url = URL(string: "https://rate.bot.com.tw/xrt?Lang=zh-TW") else {
            print("Invalid URL!")
            return
        }
        
        // use our url to customize an URL request
        let request = URLRequest(url: url)
        
        // use the customized request to create an URL session's data task, and define the completion closure
        let task = URLSession.shared.dataTask(with: request) { (data, response, error) in
            if let response = response {
                if let mimeType = response.mimeType {
                    print("mimeType: \(mimeType)")
                }
            }
            
            // if request if failed, print the error message
            if let error = error {
                print("error: \(error.localizedDescription)")
                return
            }
            
            // if data does exist, convert it to string and print out
            if let data = data {
                if let html = String(data: data, encoding: .utf8) {
                    // we have good data, go back to main thread
                    DispatchQueue.main.async {
                        print("html: \(html)")
                    }
                }
                return
            }
        }
        
        // start our URL session's data task
        task.resume()
    }
}
